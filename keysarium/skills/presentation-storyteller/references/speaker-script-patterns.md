# Speaker Script Patterns

## Core Principles

**The script should NOT:**
- Read the slide literally
- Start with «На этом слайде...»
- Use «Как вы видите...»
- List facts without context
- End abruptly without transition

**The script SHOULD:**
- Open with a hook (question, story, provocation)
- Connect to audience's reality
- Include pauses for emphasis
- Provide transitions to next slide
- End blocks with mini-summaries

---

## Opening Patterns

### 1. Rhetorical Question
```
> *«Поднимите руку, кто за последний год слышал от руководства: 
> "[quote relevant to topic]"?»*
>
> *(пауза, смотрим на реакцию)*
>
> *«Почти все. И это не [negative framing] — это [positive reframe].»*
```

### 2. Surprising Statistic
```
> *«[Big number]. Это не опечатка. [Контекст числа].»*
>
> *(пауза)*
>
> *«И знаете что самое [интригующее/тревожное]? [Follow-up insight]»*
```

### 3. "What If" Scenario
```
> *«Представьте: [vivid scenario relevant to audience].»*
>
> *«Что бы вы сделали? [Pause for mental processing]»*
>
> *«Это не гипотетика — это [real situation / near future].»*
```

### 4. Bold Claim
```
> *«[Provocative statement that challenges conventional wisdom].»*
>
> *(пауза, позволяем осознать)*
>
> *«Звучит [смело/странно/контринтуитивно], но за следующие [N] минут 
> я покажу, почему это правда.»*
```

### 5. Story Opening
```
> *«[Short anecdote — 2-3 sentences max]. Знакомо?»*
>
> *«Это история не одной компании — это [broader pattern].»*
```

## Transition Patterns

### Between Slides (same block)

**Logical continuation:**
```
> *«И это приводит нас к следующему вопросу: [question that next slide answers]»*
```

**Building on previous:**
```
> *«Но это только часть картины. Теперь посмотрим на [next aspect]»*
```

**Contrast:**
```
> *«Это была теория. Теперь — практика.»*
```

### Between Blocks

**Summary + Bridge:**
```
> *«Итак, мы разобрали [block summary]. Но возникает вопрос: 
> как это применить? Об этом — в следующем блоке.»*
```

**Emotional shift:**
```
> *«Хватит о проблемах. Давайте о решениях.»*
```

**From Problem to Solution:**
```
> *(пауза, смена тона)*
>
> *«А что если я скажу, что есть другой путь?»*
```

**From Solution to Proof:**
```
> *«Звучит хорошо в теории. Но работает ли это на практике? 
> Давайте посмотрим на цифры.»*
```

## Explanation Patterns

### For Statistics
```
> *«Давайте распакуем эту цифру. [XX]% — это каждый [Nth] из вас. 
> Или [alternative framing that makes it concrete].»*
```

### For Technical Concepts
```
> *«Если упростить: [simple analogy]. Как [everyday example], 
> только для [technical domain].»*
```

### For Comparisons
```
> *«Смотрите на [column A]. Теперь на [column B]. 
> Разница в [key differentiator] — и вот почему это критично: [reason].»*
```

### For Processes
```
> *«Представьте типичный день. [Scenario]. Что происходит дальше?»*
>
> *«[Step 1 description]»*
>
> *«Затем [Step 2 description]»*
>
> *«И в конце [Outcome]. Без [pain point from earlier].»*
```

---

## Emphasis Patterns

### For Key Numbers
```
> *«[Number]. Запомните эту цифру. Она будет важна через [N] слайдов.»*
```

### For Important Claims
```
> *(замедляемся)*
>
> *«И вот что критически важно понять: [key point].»*
>
> *(пауза)*
```

### For Counterintuitive Points
```
> *«Это может показаться странным, но [counterintuitive claim]. 
> Позвольте объяснить, почему.»*
```

---

## Objection Handling Patterns

### Preemptive
```
> *«Вы можете подумать: "[common objection]". Справедливо. Но [reframe].»*
```

### Acknowledgment + Pivot
```
> *«Да, [valid concern]. И именно поэтому мы [how solution addresses it].»*
```

### Evidence-Based
```
> *«Когда мы слышим "[objection]", мы показываем [evidence/case study]. 
> Результат: [outcome].»*
```

---

## Closing Patterns

### For Blocks
```
> *«Три ключевых вывода из этого блока:*
> *Первое — [takeaway 1].*
> *Второе — [takeaway 2].*
> *Третье — [takeaway 3].»*
```

### For Full Presentation
```
> *«Давайте подведём итог.»*
>
> *«[Главное сообщение — одно предложение].»*
>
> *«[Напоминание ключевой ценности].»*
>
> *«[CTA — конкретное действие].»*
>
> *«Вопросы?»*
```

### Call-to-Action Patterns
```
> *«У вас есть три варианта. [Option 1] — если [condition]. 
> [Option 2] — если [condition]. [Option 3] — для тех, кто [condition].»*
>
> *«Моя рекомендация: [specific option]. Потому что [reason].»*
```

## Timing Guidelines

| Content Type | Words per Minute | Example |
|--------------|------------------|---------|
| Normal pace | 120-150 | Most content |
| Key points | 80-100 | Slow down for emphasis |
| Stories | 140-160 | Slightly faster |
| Data/numbers | 100-120 | Slower for comprehension |

**Rule of thumb:** 
- 1 slide = 1.5-2 minutes for content slides
- 1 slide = 30-45 seconds for transition slides
- Add 20% buffer for audience reactions

## Anti-Patterns to Avoid

### ❌ Reading the Slide
```
BAD: «На этом слайде вы видите три столпа нашего решения: 
      цена, безопасность и удобство.»
```
```
GOOD: «Наше решение строится на трёх принципах. 
       Первый — и самый важный для CFO — это экономика...»
```

### ❌ Apologizing
```
BAD: «Извините за сложный слайд, но...»
```
```
GOOD: «Этот слайд требует внимания. Разберём по шагам.»
```

### ❌ Filler Phrases
```
BAD: «Ну, в принципе, если говорить в целом...»
```
```
GOOD: [Direct statement without filler]
```

### ❌ Ending with "Questions?"
```
BAD: «...и это всё. Вопросы?»
```
```
GOOD: «...и это означает [final benefit]. 
       Я готов ответить на вопросы.»
```

### ❌ Monotone Delivery Indicators
**Signs your script enables monotone:**
- All sentences same length
- No pauses indicated
- No emotional beats marked
- No questions to audience

## Script Markup Conventions

| Markup | Meaning |
|--------|---------|
| *(пауза)* | 1-2 second pause |
| *(длинная пауза)* | 3-4 second pause |
| *(смотрим на аудиторию)* | Eye contact moment |
| **bold text** | Emphasize vocally |
| *«quoted text»* | Direct speech / what to say |
| *(замедляемся)* | Slow down pace |
| *(меняем тон)* | Shift emotional register |
| [Stage direction] | Physical action |

## Language Guidelines for Russian

### Use:
- «Вы» (formal) for business presentations
- Active voice: «Мы делаем» not «Нами делается»
- Concrete nouns: «деньги» not «финансовые ресурсы»
- Numbers in spoken form: «триста тысяч» not «300K»

### Avoid:
- Excessive Anglicisms when Russian equivalent exists
- Complex subordinate clauses (split into sentences)
- Abstract nouns: «оптимизация» → «мы ускоряем»
- Passive constructions

### Power Words (Russian):
- «Представьте» (triggers imagination)
- «Знакомо?» (creates connection)
- «Факт:» (signals evidence)
- «Критически» (emphasizes importance)
- «Именно поэтому» (creates causality)
- «Не X, а Y» (creates contrast)
