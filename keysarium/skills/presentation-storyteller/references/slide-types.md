# Slide Types Reference

## Quick Reference

| Type | Purpose | Max Content | Time |
|------|---------|-------------|------|
| Title | Hook, promise | 10 words + subtitle | 30s |
| Agenda | Roadmap | 4-6 items | 1 min |
| Problem | Amplify pain | 3-5 pain points | 1.5-2 min |
| Statistics | Evidence | 1-3 key numbers | 1 min |
| Dilemma | False choice | 2 options | 1.5 min |
| Solution | Core value | 3-4 pillars | 2 min |
| Product | What we offer | 4-6 features | 2 min |
| Technical | Deep dive | Diagram + bullets | 2 min |
| Comparison | Differentiation | Table/chart | 1.5 min |
| Proof | Evidence | Case/TCO | 2 min |
| CTA | Next steps | 2-3 options | 1.5 min |
| Contact | Close | Details + QR | 30s |

---

## 1. Title Slide

**Purpose:** Hook audience, make a promise

**Template:**
```markdown
## СЛАЙД 1: [Заголовок — bold claim]

**Визуал:**
- Крупный заголовок: **«[Провокационное утверждение]»**
- Подзаголовок: *[Конкретизация или вопрос]*
- Логотип компании (нижний правый угол)
- Фоновое изображение (опционально, не отвлекающее)

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«[Открывающий вопрос к аудитории или провокационное утверждение]»*
>
> *(пауза, смотрим на реакцию)*
>
> *«[Обещание: что аудитория получит за следующие N минут]»*
```

**Good Examples:**
- «Гибридный AI: Контроль данных без потери скорости»
- «Почему 90% ML-проектов не доходят до production»
- «Zero to Production за 30 дней: миф или реальность?»

**Bad Examples:**
- «Презентация компании X» ❌
- «Обзор продуктов и услуг» ❌
- «Итоги Q3 2025» ❌

---

## 2. Agenda Slide

**Purpose:** Set expectations, create roadmap

**Template:**
```markdown
## СЛАЙД 2: Повестка дня

**Визуал:**
```
1. 🔥 [Блок 1 — интригующее название]
2. 🏛️ [Блок 2 — название]  
3. 🛠️ [Блок 3 — название]
4. 💰 [Блок 4 — название]
5. 🚀 [Блок 5 — название]
```

**Внизу:** *«[N] минут + время на вопросы»*

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Вот наш маршрут на ближайшие [N] минут. [Краткое описание каждого блока 
> и почему он важен для аудитории]»*
>
> *«В конце — конкретные цифры и конкретные шаги. Договорились?»*
```

**Guidelines:**
- 4-6 пунктов максимум
- Используйте emoji для визуального разделения
- Названия блоков должны интриговать, не описывать

---

## 3. Problem Slide

**Purpose:** Amplify pain, create urgency

**Template:**
```markdown
## СЛАЙД N: [Название проблемы]

**Визуал:**
Таблица сравнения или список болей:

| 😰 **Вариант A** | 😰 **Вариант B** |
|------------------|------------------|
| [Боль 1] | [Боль 1] |
| [Боль 2] | [Боль 2] |
| [Боль 3] | [Боль 3] |

**Внизу:** ~~«[Ложная дилемма]»~~ → ✅ *«[Правильный путь]»*

**📚 ИСТОЧНИКИ:**
1. [Источник]: [URL]

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Давайте честно. [Описание ситуации, в которой находится аудитория]»*
>
> *«[Описание боли варианта A]»*
>
> *«[Описание боли варианта B]»*
>
> *(пауза)*
>
> *«А что если я скажу, что выбирать не нужно?»*
```

**Emotional Techniques:**
- Используйте «вы» — персонализация
- Конкретные цифры боли (стоимость, время, риск)
- Риторические вопросы
- Пауза перед решением

---

## 4. Statistics Slide

**Purpose:** Provide evidence, build credibility

**Template:**
```markdown
## СЛАЙД N: [Заголовок с ключевым инсайтом]

**Визуал:**
Крупные цифры с контекстом:

```
   [XX]%              [YY]%              [ZZ]%
[Метрика 1]        [Метрика 2]        [Метрика 3]
```

**Внизу:** *«[Вывод из статистики]»*

**📚 ИСТОЧНИКИ:**
1. [Research Firm / Report Name]: [URL]

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Посмотрите на эти цифры. [Объяснение первой метрики]»*
>
> *«[Контекст второй метрики]»*
>
> *«Что это означает? [Вывод, релевантный для аудитории]»*
```

**Guidelines:**
- Максимум 3 ключевые цифры на слайд
- Каждая цифра должна иметь источник
- Контекстуализируйте: «это в 5 раз больше, чем...»

---

## 5. Solution Overview Slide

**Purpose:** Present your approach, create framework

**Template:**
```markdown
## СЛАЙД N: [Название решения / фреймворка]

**Визуал:**
Три столпа или элементы:

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│    СТОЛП 1      │  │    СТОЛП 2      │  │    СТОЛП 3      │
│  [Название]     │  │  [Название]     │  │  [Название]     │
│                 │  │                 │  │                 │
│  • [Benefit 1]  │  │  • [Benefit 1]  │  │  • [Benefit 1]  │
│  • [Benefit 2]  │  │  • [Benefit 2]  │  │  • [Benefit 2]  │
└─────────────────┘  └─────────────────┘  └─────────────────┘
```

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Наше решение строится на трёх принципах. [Краткое описание каждого]»*
>
> *«[Столп 1] — потому что [причина важности для аудитории]»*
>
> *«[Столп 2] — это означает [конкретная выгода]»*
>
> *«[Столп 3] — и это критично, потому что [связь с болями]»*
```

**Guidelines:**
- 3-4 элемента максимум (правило магических чисел)
- Каждый элемент = одна идея
- Связывайте с ранее озвученными болями

---

## 6. Technical/Product Slide

**Purpose:** Show how it works, build confidence

**Template:**
```markdown
## СЛАЙД N: [Название компонента/функции]

**Визуал:**
Диаграмма архитектуры или workflow:

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ Input   │───▶│ Process │───▶│ Output  │
└─────────┘    └─────────┘    └─────────┘
```

**Ключевые характеристики:**
- [Характеристика 1]: [значение]
- [Характеристика 2]: [значение]
- [Характеристика 3]: [значение]

**📚 ИСТОЧНИКИ:**
1. [Vendor docs / Benchmark]: [URL]

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Давайте разберём, как это работает. [Описание входа]»*
>
> *«[Описание процесса — что происходит внутри]»*
>
> *«На выходе получаем [результат]. Это важно, потому что [связь с бизнес-ценностью]»*
```

**Guidelines:**
- Один процесс/компонент на слайд
- Диаграммы > текст
- Связывайте технические детали с бизнес-ценностью

---

## 7. Comparison Table Slide

**Purpose:** Differentiate, help decision

**Template:**
```markdown
## СЛАЙД N: Сравнение [вариантов]

**Визуал:**

| Критерий | Вариант A | Вариант B | **Наше решение** |
|----------|-----------|-----------|------------------|
| [Критерий 1] | 🟡 [value] | 🔴 [value] | 🟢 **[value]** |
| [Критерий 2] | 🔴 [value] | 🟡 [value] | 🟢 **[value]** |
| [Критерий 3] | 🟡 [value] | 🟢 [value] | 🟢 **[value]** |

**📚 ИСТОЧНИКИ:**
1. [Source]: [URL]

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Давайте сравним варианты по ключевым критериям.»*
>
> *«[Критерий 1] — здесь [наше решение] выигрывает, потому что [причина]»*
>
> *«Обратите внимание на [критерий N] — это часто упускают из виду»*
```

**Guidelines:**
- 5-7 критериев максимум
- Используйте цветовую кодировку (🟢🟡🔴)
- Честно показывайте, где вы НЕ лидер

---

## 8. TCO/Proof Slide

**Purpose:** Provide evidence, justify investment

**Template:**
```markdown
## СЛАЙД N: Экономика решения

**Визуал:**
Расчётная таблица:

| | Год 1 | Год 2 | Год 3 | **ИТОГО** |
|---|---:|---:|---:|---:|
| **[Вариант A]** | $X | $Y | $Z | **$Total** |
| **[Наше решение]** | $X | $Y | $Z | **$Total** |

**Внизу крупно:** **Экономия: [XX]% за [N] лет**

**📚 ИСТОЧНИКИ:**
1. [Pricing source]: [URL]
2. [Benchmark]: [URL]

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Теперь — цифры. Потому что в конечном счёте это бизнес-решение.»*
>
> *«[Описание сценария расчёта]»*
>
> *«Разница — [сумма]. На эти деньги можно [конкретный пример]»*
```

## 9. CTA Slide

**Purpose:** Drive action, reduce friction

**Template:**
```markdown
## СЛАЙД N: Следующие шаги

**Визуал:**
```
┌─────────────────────────────────────────────────────────────────┐
│   🔍 [Вариант 1]         🧪 [Вариант 2]         🚀 [Вариант 3]  │
│   [Срок]                 [Срок]                 [Срок]          │
│                                                                 │
│   • [Что получит]        • [Что получит]        • [Что получит] │
│   • [Что получит]        • [Что получит]        • [Что получит] │
│                                                                 │
│   [Цена/условия]         [Цена/условия]         [Цена/условия]  │
└─────────────────────────────────────────────────────────────────┘
```

**Рекомендация:** *«[Какой вариант рекомендуете и почему]»*

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Три варианта — выберите комфортный.»*
>
> *«[Вариант 1] — если [условие]. [Вариант 2] — если [условие].»*
>
> *«Моя рекомендация: начните с [вариант]. Потеряете [риск], получите [выгода].»*
```

**Guidelines:**
- 2-3 варианта (не больше)
- Конкретные сроки и условия
- Рекомендуйте один вариант
- Снижайте perceived risk

---

## 10. Contact/Closing Slide

**Purpose:** Enable follow-up, memorable close

**Template:**
```markdown
## СЛАЙД N: [Финальное сообщение]

**Визуал:**
```
                    [КЛЮЧЕВОЙ TAKEAWAY]

         [QR-код на форму/landing]
         
              [Призыв к действию]

   📧 [email]         📞 [телефон]         🌐 [website]
   
   [Имя спикера] | [Должность] | [LinkedIn]
```

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Давайте подведём итог. [Краткое резюме ключевого сообщения]»*
>
> *«[Напоминание главной ценности]»*
>
> *«[CTA] — QR на экране. Или напишите мне после — контакты здесь.»*
>
> *«Вопросы?»*
```

## Slide Density Guidelines

| Slide Type | Max Words | Max Bullets | Max Data Points |
|------------|-----------|-------------|-----------------|
| Title | 15 | 0 | 0 |
| Agenda | 30 | 6 | 0 |
| Problem | 50 | 5 | 3 |
| Statistics | 20 | 0 | 3 |
| Solution | 40 | 4 per pillar | 0 |
| Technical | 60 | 5 | 5 |
| Comparison | 30 + table | 0 | 7 rows max |
| Proof/TCO | 30 + table | 3 | 5 rows max |
| CTA | 40 | 3 per option | 0 |
| Contact | 20 | 0 | 0 |
