# Example: Hybrid AI Infrastructure Presentation

**This is a complete example** of a presentation created with the presentation-storyteller skill. Use as a reference for structure, source citation, and speaker script style.

---

## Metadata

- **Тема:** Гибридный подход к AI-инфраструктуре
- **Аудитория:** CIO, CTO, IT Directors российских enterprise-компаний
- **Цель:** Запросить Quick Test / Pilot проект
- **Хронометраж:** 25-30 мин + Q&A
- **Storytelling Arc:** Problem-Solution
- **Дата:** Январь 2025

# ПЛАН ПРЕЗЕНТАЦИИ С ТЕЗИСАМИ

## Структура

### БЛОК 1: ПРОБЛЕМАТИКА (3 слайда, ~5 мин)

#### Слайд 3: Дилемма CIO/CDO
**Тип:** Problem
**Тезисы:**
- Публичное облако = потеря контроля над данными + vendor lock-in + риск US CLOUD Act
- Полный on-prem = высокий CAPEX ($200-400K за сервер) + низкая утилизация (30-50% простоя)
- Ложная дилемма «или/или» → гибрид даёт «и/и»

#### Слайд 4: Регуляторное давление 2025
**Тип:** Statistics + Timeline
**Тезисы:**
- 01.01.2025: 187-ФЗ — запрет иностранного ПО на КИИ
- 30.05.2025: Оборотные штрафы за утечки до 500 млн руб
- 01.07.2025: 152-ФЗ — первичный сбор ПДн только в РФ
- **Вывод:** On-prem компонент — теперь не опция, а требование закона

### БЛОК 2: РЕШЕНИЕ (3 слайда, ~5 мин)

#### Слайд 6: Столп #1 — ЦЕНА
**Тип:** Solution Pillar
**Тезисы:**
- H100: $25-40K за карту, 8-GPU сервер $200-400K
- 75% организаций не загружают GPU выше 70%
- Гибрид: критичное на своём железе + эластичное в облаке
- **Вывод:** TCO гибрида на 40-50% ниже dedicated

## СЛАЙД 10: HAMI Shared GPU

**Визуал:**
Сравнительная таблица:

| | MIG | Time-slicing | **HAMI** |
|---|---|---|---|
| Гранулярность | 7 партиций макс | Нет лимитов | **Любой размер** |
| Изоляция | Hardware | Нет | **Soft + limits** |
| Перенастройка | Reboot | — | **Динамическая** |
| Поддержка GPU | Только Ampere+ | Любые | **Любые NVIDIA** |

**Внизу:** *«HAMI — CNCF Sandbox проект, 50+ компаний в production»*

**📚 ИСТОЧНИКИ:**
1. OpenMetal — MIG vs Time-Slicing: https://openmetal.io/resources/blog/mig-vs-time-slicing-gpu-sharing/
2. Project HAMi GitHub: https://github.com/Project-HAMi/HAMi
3. InfraCloud — GPU Sharing Guide: https://medium.com/infracloud-technologies/guide-to-gpu-sharing-techniques-vgpu-mig-and-time-slicing

---

**🎤 КАК РАССКАЗЫВАТЬ:**

> *«Давайте нырнём в технологию. Одна из главных проблем GPU-инфраструктуры — как делить карту между задачами.*
>
> *NVIDIA предлагает MIG — Multi-Instance GPU. Красиво звучит, но: максимум 7 партиций, фиксированные размеры, и чтобы перенастроить — нужен reboot.*
>
> *Есть time-slicing — просто по очереди даём доступ. Дёшево и сердито, но никакой изоляции. Одна задача сожрала всю память — остальные падают.*
>
> *(пауза, указываем на HAMI)*
>
> *HAMI — другой подход. Делим карту на любые доли: хотите 1GB — пожалуйста. При этом есть изоляция памяти. И перенастройка — динамическая, без перезагрузки.*
>
> *Это проект CNCF Sandbox, 50+ компаний используют в production.»*

## Notes on This Example

**What makes this presentation effective:**

1. **Problem-Solution arc** — Clear tension building before relief
2. **Verified sources** — Every statistic has a URL
3. **Audience-specific language** — Speaks to CIO/CTO concerns
4. **Concrete numbers** — $400K, 30%, 500M RUB
5. **Actionable CTA** — Three clear options with recommendation
6. **Speaker scripts** — Include pauses, questions, transitions
7. **Visual descriptions** — Clear guidance for slide design
